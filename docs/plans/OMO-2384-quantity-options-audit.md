# OMO-2384 — 제품 옵션 수량 드롭다운 전수조사 + 동일가격 프로모션 UI

## 보드 요청 (원문)
> 예를들어 명함이면 지금 100 200 500개가 똑같고 2000다음 250개 이고 뭐 이런 에러들 있거든 전수조사해서 명확하게 세팅해.
> 100, 200개 가격과 500개 가격이 똑같다 프로모션중이다 이런 식으로 진행하면 어떨까. 버튼이나 옵션 주위 눈에 띄는 곳에 메세지 좀 날려주고 같다고 좀 티내는 거.

## 근본 원인 (Swadpia 가격 매트릭스의 최소 수량 한계)

`ProductConfigurator.tsx`의 `lookupSwadpiaCost`는 **요청한 수량을 못 찾으면 다음 상위 수량의 단가를 그대로 반환**한다 (≥quantity 검색). 그 결과 고객은 서로 다른 수량 옵션을 골라도 같은 가격을 본다.

### 카테고리별 Swadpia 실제 단가 매트릭스 확인 (2026-06-05 기준)

| Slug | Swadpia 카테고리 | print_info1 매트릭스 | 종이별 실제 수량 키 |
|------|-----------------|---------------------|----------------------|
| business-cards | CNC1000 | 있음 | SNW250W00: 500 / 1,000 / 1,500 / 2,000 / 3,000 / ... (최소 500)<br>SNW300W00: 200 / 400 / 600 / 800 / 1,000 / ... (최소 200) |
| premium-business-cards | CNC2000 | 있음 | 모든 paper: 200 / 400 / 600 / 800 / 1,000(=0원 비정상) |
| stickers | CST1000 | **없음** (paper_info만 존재) | DB extra_price_krw 폴백 |
| die-cut-stickers | CST2000 | **없음** | DB 폴백 |
| flyers | CLF1000 | **없음** | DB 폴백 |
| brochures | CLF2000 | 있음 | 다양 |
| postcards | CDP3000 | **없음** | DB 폴백 |
| posters | CPR2000 | 있음 (단, paper_code=빈 문자열) | 매칭 실패 → DB 폴백 |
| banners | CPR5000 | **없음** | DB 폴백 |

### 명함의 100/200/500 가격이 똑같은 이유
- 기본 paper_code가 SNW250W00 인데 Swadpia 최소 수량이 500.
- 100 → 다음 상위 500 단가(3,000원) 반환
- 200 → 다음 상위 500 단가(3,000원) 반환
- 500 → exact 매칭(3,000원)
- → 3가지 옵션이 모두 똑같이 보임. (실제로 100·200 주문 들어와도 우리는 Swadpia에 최소 500장 발주해야 함 → 그게 비즈니스 사실)

### 프리미엄 명함 1000장 = $200으로 비정상 점프
- CNC2000 매트릭스에서 qty=1000 의 print_unit2 가 **0**(Swadpia 측 등록 누락 또는 "전화문의" 영역)
- `costKrw > 0` 가드 때문에 폴백 → DB extra_price_krw(50,000원) + base_price(30,000) × 3.3 = 26.4만원 ≈ $200
- 200/500 가 $8~$26 인데 1000 만 $200 → 가격 점프

## 처리 방안

### A. UI — "동일가격 프로모션" 배지 (이번 PR)
`ProductConfigurator.tsx`의 수량 옵션 렌더링에:
1. 각 수량별 Swadpia 실효 단가를 미리 계산
2. 동일 단가로 묶이는 그룹을 찾고, 가장 큰 수량을 "기준 수량"으로 잡음
3. 기준 수량보다 작은 옵션에는 `🎁 SAME PRICE PROMO — N장 가격으로` 배지 표시
4. 가격 요약 박스에는 "💡 100매도 500매 가격으로 — 추가 매수 무료 프로모션" 메시지

이 방식의 장점:
- 가격이 똑같이 보이는 혼란을 마케팅 기회로 전환
- "추가 매수 무료" 프레이밍 → 고객은 500 선택 유도됨 (Swadpia 발주 최소수량과도 일치)
- Swadpia 매트릭스 변동에 자동 대응 (배지가 자동 노출/숨김)

### B. 데이터 — Swadpia 매트릭스 비정상 케이스 핸들
1. `lookupSwadpiaCost` 가 `print_unit2 === 0` 인 항목을 건너뛰도록 수정.
   - 프리미엄 명함 1000 → 800 단가 사용 (24,000원 × 3.3 ≈ $25)
   - 더 이상 0→$200 점프 없음

### C. 데이터 — 옵션 수량 명시적 정렬 (확정 후 별도 마이그레이션)
이번 heartbeat 에서는 데이터 변경 없음. 옵션 수량 자체를 변경하면 기존 URL/링크/이메일이 깨지므로 보드 승인 후 별도 진행.
